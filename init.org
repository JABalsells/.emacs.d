* About

In Emacs, you can use an init.org file instead of (or alongside) the usual =~/.emacs= for your configuration. This approach is called /literate configuration/: you write your setup in an Org file with explanations, and Emacs uses =org-babel-tangle= to extract the code blocks into a separate file—your =init.el=. This way, your configuration is both executable and documented for your future self (and anyone else who looks at it).


Since Emacs loads ~init.el~ automatically, you just need a small ~.emacs~ that loads the Org file. Put this in ~~/.emacs~:

#+BEGIN_SRC emacs-lisp :tangle no
  (org-babel-load-file (expand-file-name "init.org"
					 user-emacs-directory))
#+END_SRC

  ;; Enable major package archives
  (require 'package)

  ;; Automatically install use-package the first time
  (unless (package-installed-p 'use-package)
    (package-refresh-contents)
    (package-install 'use-package))

  (setq use-package-always-ensure  t   ; fetch if missing

** GUI-only frame adjustments:

*vertical border* The thin line is Emacs’s vertical border face (shown between side-by-side windows). This sets the border color equal to the background, so it disappears in GUI frames.

*fringes* Fringes are the thin margins at the left/right edges of GUI Emacs windows where continuation/line-wrap and breakpoint indicators show up.

In terminal (TTY) Emacs, fringes don't exist at all. The terminal can only render the text area. So Emacs just uses the first/last text column to display wrap indicators (=\=,=>=,=<=, etc.). 


#+begin_src emacs-lisp
  (when (display-graphic-p)
    (set-face-attribute 'vertical-border nil 
			:foreground (face-background 'default))
    (add-to-list 'default-frame-alist '(undecorated . t))
    (scroll-bar-mode 0)
    (fringe-mode 10)) ;; Set left/right fringe width 0,-1,20 (pixels)

#+end_src

First, a helper function that detects if point is inside math mode.
This is helpful for entering math snippets.
#+begin_src emacs-lisp

  (defun my-tex-in-math-p ()
    "Return non-nil if point is inside LaTeX math.
    Works with AUCTeX and (best-effort) Org."
    (cond
     ;; AUCTeX (older/newer names)
     ((fboundp 'texmathp) (texmathp))
     ((fboundp 'TeXmathp) (TeXmathp))
     ;; Org-mode LaTeX fragments
     ((and (derived-mode-p 'org-mode)
	   (fboundp 'org-inside-LaTeX-fragment-p))
      (org-inside-LaTeX-fragment-p))
     (t nil)))

  (defun my-wrap-math (s)
    "Wrap S in \\( ... \\) unless already in math."
    (if (my-tex-in-math-p) s (concat "\\(" s "\\)")))  

     #+end_src

** PDF Tools

In AUCTeX (the LaTeX package for Emacs), =C-c C-a= (=TeX-command-run-all=) runs the full compile -> view -> bibtex -> etc. chain. By default, it asks you to save if the buffer is modified. You can tell it to always save without prompting before running any TeX command.

#+begin_src emacs-lisp
  ;; Always save the current buffer before compiling
  (setq TeX-save-query nil)
  (setq TeX-ask-before-save nil)
#+end_src

A few details:


* =TeX-save-query=: if non-nil (the default), AUCTeX asks before saving. Setting it to =nil= disables the query and just saves.

* =TeX-ask-before-save=: an older/related variable; also set to =nil= to be safe.

So with these set, hitting =C-c C-a= will save your =.tex= file automatically and proceed with the compile sequence without asking.
